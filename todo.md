## 验收器

### 实现验收器功能

- 标记：S16；难度 D4；范围 R4；建议分支 `feature/acceptor`；依赖结构匹配规则明确，建议在基础模拟冲突规则后做。

- 目标：实现验收器对材料结构的验收逻辑。
- 要点：
  - 相连的验收器必须验收相连的材料结构。
  - 被验收结构不能多方块，也不能少方块；必须与相连验收器期望的结构完全匹配。
  - 需要定义验收器之间如何组成验收组，以及验收组如何映射到材料结构。
  - 验收失败时应保留清晰的调试或 UI 反馈。
- 验收：
  - 多个相连验收器只验收一个对应的相连结构。
  - 材料结构少一块、多一块、断开、额外连接时都不会误判通过。

## 传送器

### 实现传送器功能

- 标记：S11；难度 D3；范围 R3；建议分支 `feature/teleporter-material`；可与玩家视角传送分支并行调研，但应先合并材料传送核心。

- 目标：完善传送入口 / 出口对材料结构的传送逻辑。
- 要点：
  - 明确入口和出口配对方式。
  - 传送入口一次只传送入口位置上的单个材料方块，而不是保留整个焊接结构。
  - 如果三个方块焊在一起，只传送其中一个方块时，剩余两个方块留在原地，原有焊接关系需要断开或清理。
  - 如果三个方块都分别被传送到另一边，它们在出口侧也应是断开的状态，不会自动保留或恢复焊接关系。
  - 传送前检查出口目标格是否可放置被传送方块。
  - 传送失败时不改变世界状态。
  - 传送材料方块时可以保留该方块自身的面标记，但不能保留与其他方块的焊接关系。
- 验收：
  - 配对有效且出口目标格可用时，入口位置的单个材料方块正确传送。
  - 传送后原结构和出口侧方块都不会残留错误焊接关系。
  - 出口目标格被阻挡时，点击或模拟触发都无效且无副作用。

### 点击传送器传送玩家视角

- 标记：S12；难度 D3；范围 R3；建议分支 `feature/teleporter-player-view`；依赖传送器配对和出口可用性检查。

- 目标：在游玩期和模拟期，允许玩家点击传送器后传送自己的视角。
- 要点：
  - 玩家点击入口时，如果存在有效出口，则尝试传送玩家视角。
  - 玩家传送后，相对入口位置的偏移应等于传送后相对出口位置的偏移。
  - 传送成功后，玩家看到被点击的方块位置关系尽量保持不变，只是周围场景变化。
  - 传送前必须检查出口位置是否可以放置玩家。
  - 如果出口位置不可放置玩家，点击无效。
- 验收：
  - 游玩期和模拟期都可以点击传送器传送视角。
  - 传送后玩家相对入口 / 出口的空间偏移一致。
  - 出口空间不足时不会传送。

  - 可以使用一个模型面片实现目标面高亮。
  - 高亮应贴合被瞄准方块的面，并随瞄准面变化。
  - 高亮不能遮挡放置预览，也不能造成深度闪烁。
- 验收：
  - 玩家能清楚看出当前会往哪个面放置方块。
  - 不同朝向、不同距离下高亮稳定。

### 修复定制模型方块半透明预览

- 标记：S06；难度 D2；范围 R2；可在 `main` 直接执行；与放置目标面高亮都在放置 / 渲染预览附近，建议连续做。

- 目标：修复传送带、活塞、钻头、焊接器等定制模型方块放置时半透明预览不生效的问题。
- 要点：
  - 检查这些方块的定制模型渲染路径是否绕过了预览透明材质。
  - 确认 `PartsOnly`、多部件模型、派生头部模型等情况是否都应用透明材质。
  - 同时检查旋转和朝向预览是否正确。
- 验收：
  - 传送带、活塞、钻头、焊接器放置预览都正确半透明。
  - 普通方块和其他定制模型方块预览不回归。
